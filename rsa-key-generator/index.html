What is RSA Asymmetric Cryptography?

RSA (Rivest–Shamir–Adleman) is a public-key cryptosystem widely used for secure data transmission. In an asymmetric cryptographic system, two different mathematically linked keys are generated: a Public Key and a Private Key.

The Public Key can be shared with anyone. It is used to encrypt messages. However, once a message is encrypted with the Public Key, it can only be decrypted by the matching Private Key. The Private Key must be kept absolutely secret. RSA is the foundational technology behind HTTPS/TLS certificates, SSH logins, and JSON Web Token (JWT) RS256 signing.

Why use the Web Crypto API for RSA Generation?

Traditionally, developers use command-line tools like OpenSSL to generate RSA keys. However, relying on online, third-party key generators is extremely dangerous. If a remote server generates your Private Key, the server owner can silently save a copy of it, completely compromising your system's security.

This ZeroKey Utility solves this problem by using the native window.crypto.subtle.generateKey() API. The keys are mathematically generated using your device's local entropy (CPU randomness). The Private Key never leaves your browser, ensuring absolute zero-knowledge security.

What is PEM Format?

PEM (Privacy-Enhanced Mail) is the most common format for X.509 certificates and cryptographic keys. It is essentially a Base64 encoded string of the raw binary key, wrapped with explicit header and footer lines (e.g., -----BEGIN PRIVATE KEY-----). This text-based format allows cryptographic keys to be easily copied, pasted, and stored in `.env` files or CI/CD pipelines without binary corruption.
