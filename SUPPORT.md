# Getting Support for ZeroKey

Thank you for using ZeroKey! We want to make sure you have the best experience possible while keeping your digital privacy completely intact. 

Please review the following support channels to find the right place for your question or issue.

## 🚨 Critical Security Vulnerabilities
**Do NOT open a public GitHub issue for security flaws.**

If you have discovered a vulnerability in the encryption pipeline, key derivation, or backend infrastructure, please refer to our [Security Policy](SECURITY.md) immediately. 

Report all security issues directly via:
* **Email:** [Dippan.connect@gmail.com](mailto:Dippan.connect@gmail.com)
* **Secure Form:** [ZeroKey Contact Page](https://zerokey.vercel.app/contact)

*We prioritize security reports above all other support requests.*

## 🐛 Bug Reports & Feature Requests
If you have found a glitch in the UI, a browser compatibility issue, or have a brilliant idea for a new feature, GitHub Issues is the place to go.

* **Search First:** Before opening a new issue, please search the [Existing Issues](https://github.com/kdippan/zerokey/issues) to see if someone else has already reported it.
* **Open an Issue:** If your issue is new, open a new issue. Please include:
  * Your operating system and browser (e.g., iOS 17 Safari, Windows 11 Chrome).
  * Clear steps to reproduce the bug.
  * Screenshots (if applicable).

## 📖 General Questions & Troubleshooting
If you are confused about how the client-side Web Crypto API works, or how the URL fragment handles the decryption keys, please check our official documentation first:

* **[Technical Architecture & FAQ](https://zerokey.vercel.app/about)**
* **[Terms & Conditions](https://zerokey.vercel.app/terms)**

If your question isn't answered there, feel free to reach out via the [Contact Page](https://zerokey.vercel.app/contact).

## ⏱️ Support Expectations
ZeroKey is a free, open-source project currently developed and maintained by a student developer ([@kdippan](https://github.com/kdippan)). 

While I strive to fix bugs and answer questions as quickly as possible, please understand that response times may vary depending on my academic schedule. **Security vulnerabilities will always receive immediate attention.**
