How to Encrypt Files in the Browser Using the Web Crypto API

March 16, 2026
Dippan Bhusal
10 Min Read

Uploading raw, unencrypted files directly to an S3 bucket or a cloud database is a massive liability. If you are building a secure messaging app, a whistleblower portal, or a document-sharing platform, you must assume your servers will eventually be compromised.

The safest approach is client-side encryption. By encrypting the file in the user's browser before it ever hits the network, you ensure that your backend only stores unreadable ciphertext. In this tutorial, we will use Vanilla JavaScript and the native window.crypto.subtle API to encrypt a file using AES-256-GCM.

Why Use the Web Crypto API?

In the past, developers relied on heavy external libraries like Crypto-JS to handle browser cryptography. Today, the Web Crypto API is built directly into all modern browsers. It is significantly faster because it has access to the device's underlying hardware acceleration, and it introduces zero supply-chain vulnerabilities because there are no NPM packages to install.

Step 1: Reading the File as an ArrayBuffer

Cryptographic functions do not understand standard DOM File objects. Before we can encrypt a file, we must convert it into raw binary data using an ArrayBuffer. 

const fileInput = document.getElementById('myFileInput');

fileInput.addEventListener('change', async (event) => {
    const file = event.target.files[0];
    if (!file) return;

    // Read the file as raw binary data
    const arrayBuffer = await file.arrayBuffer();
    
    // Proceed to encryption...
    await encryptMyFile(arrayBuffer);
});

Step 2: Generating a Cryptographic Key

For this example, we will generate a fresh AES-256 symmetric key. In a real-world application, you might derive this key from a user's password using PBKDF2, but for simplicity, we will let the browser generate a secure random key. 

async function generateAESKey() {
    return await window.crypto.subtle.generateKey(
        {
            name: "AES-GCM",
            length: 256
        },
        true, // Allow the key to be exported later if needed
        ["encrypt", "decrypt"] // Key usage
    );
}

Step 3: Encrypting the ArrayBuffer

AES-GCM (Galois/Counter Mode) is the industry standard for symmetric encryption because it provides both confidentiality and data authenticity. It requires an Initialization Vector (IV)—a piece of random data that ensures the same file encrypted twice will yield different ciphertext. 

async function encryptMyFile(fileBuffer) {
    const key = await generateAESKey();
    
    // Generate a 12-byte random Initialization Vector (IV)
    const iv = window.crypto.getRandomValues(new Uint8Array(12));

    // Encrypt the file data
    const encryptedBuffer = await window.crypto.subtle.encrypt(
        {
            name: "AES-GCM",
            iv: iv
        },
        key,
        fileBuffer
    );

    console.log("File successfully encrypted locally!");
    
    // At this point, you would send 'encryptedBuffer' and 'iv' 
    // to your backend, NOT the original file.
    return { encryptedBuffer, iv, key };
}

The Hard Part: Key Management

The code above successfully converts a file into secure ciphertext. However, the hardest part of building a zero-knowledge application is Key Exchange. How do you give the receiving user the decryption key without sending it to your database?

A common technique is the URL Fragment Exploit. You append the exported decryption key to the end of a shareable link after a hash symbol (e.g., https://yoursite.com/view#SECRET_KEY). Because browsers do not transmit URL fragments to the server, the backend remains completely blind to the key.

Conclusion

By shifting cryptographic operations to the client-side, you completely eliminate the liability of storing sensitive user data. The Web Crypto API makes it remarkably simple to integrate military-grade AES-256-GCM encryption directly into your frontend JavaScript applications without relying on third-party libraries.
