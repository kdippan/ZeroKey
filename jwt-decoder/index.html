What is a JSON Web Token (JWT)?

                
A JSON Web Token (JWT) is an open standard (RFC 7519) that defines a compact and self-contained way for securely transmitting information between parties as a JSON object. This information can be verified and trusted because it is digitally signed using a secret (with the HMAC algorithm) or a public/private key pair using RSA or ECDSA.

                
                
Why is pasting JWTs online dangerous?

                
Many online JWT decoders send the token you paste to their backend servers to process the decoding. If you paste a production authentication token, you are effectively handing over a live session key to a third party. A compromised token can lead to session hijacking and severe data breaches.

                
This ZeroKey JWT Decoder is designed with absolute privacy in mind. It uses vanilla JavaScript to parse and decode the Base64Url strings strictly within your local browser environment. No HTTP requests are made, and your token never touches a remote server.


                
Understanding the JWT Structure

                
A JWT consists of three parts separated by dots (.), which typically look like this: xxxxx.yyyyy.zzzzz

                
    
                    
  • Header: Consists of two parts: the type of the token (JWT) and the signing algorithm being used, such as HMAC SHA256 or RSA.
    • 
                    
  • Payload (Claims): Contains the claims. Claims are statements about an entity (typically, the user) and additional data. Do not put secret information in the payload, as it can be easily decoded by anyone who has the token.
    • 
                    
  • Signature: To create the signature part, you have to take the encoded header, the encoded payload, a secret, the algorithm specified in the header, and sign that. The signature is used to verify the message wasn't changed along the way.
