What is an HMAC?

HMAC stands for Hash-based Message Authentication Code. It is a specific type of message authentication code involving a cryptographic hash function (like SHA-256) combined with a secret cryptographic key.

When two parties share a secret key, HMACs allow them to verify both the data integrity and the authenticity of a message. If a hacker intercepts a message and modifies the payload, the HMAC signature will completely change. Because the hacker does not possess the secret key, they cannot generate a new, valid signature for their tampered payload.

Why are HMACs used for Webhooks?

Major platforms like Stripe, GitHub, Shopify, and Slack use HMAC signatures to secure their webhooks. When these platforms send an HTTP POST request to your backend server, they calculate an HMAC using the raw JSON payload and a secret key they gave you when you registered.

They attach this HMAC to a header (e.g., Stripe-Signature or X-Hub-Signature-256). Your server must take the incoming JSON, use your secret key to calculate its own HMAC, and compare it to the header. If they match, you know the webhook genuinely came from the service provider and was not forged by an attacker.

Never expose your Secret Key online

Pasting your Stripe or GitHub webhook secrets into random online calculators is incredibly dangerous. If a remote server logs your secret key, attackers can send malicious webhooks to your application (e.g., faking a successful payment). This ZeroKey Utility utilizes the native window.crypto.subtle.sign() API. All signature generation happens locally in your browser memory, ensuring your API secrets are never transmitted across the internet.
